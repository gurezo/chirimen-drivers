# Devices 画像ファイルガイドライン

## 概要

このディレクトリには、CHIRIMENドライバーでサポートされているデバイスの画像ファイルが格納されています。現在、100個の画像ファイルが含まれており、各デバイスの視覚的な識別を支援するために使用されます。

## 命名規則

### 基本原則

画像ファイル名は、対応するデバイスの型番または一般的な名称に基づいて命名されます。

### 命名パターン

#### 1. デバイス型番ベース（推奨）
デバイスの型番をそのまま使用します。

**良い例:**
- `BME280.jpg` - デバイス型番をそのまま使用
- `ADS1015.jpg` - 型番ベース
- `HT16K33.jpg` - 型番ベース

#### 2. 大文字小文字の扱い
- **デバイス型番**: 型番の表記に従う（通常は大文字）
  - 例: `A4988.jpg`, `BME280.jpg`, `MPU6050.jpg`
- **一般的な名称**: 小文字で始める
  - 例: `fan.jpg`, `camera.jpg`, `LED.jpg`

#### 3. 区切り文字の使用

複数の単語や要素を組み合わせる場合、以下の区切り文字を使用します：

- **アンダースコア (`_`)**: デバイス型番のバリエーションや複合デバイスを表す場合
  - 例: `HT16K33_7seg.jpg`, `ENS160_AHT20.jpg`, `RFID2_WS1850S.jpg`
- **ハイフン (`-`)**: ブランド名やシリーズ名を含む場合
  - 例: `Grove-Light.jpg`, `M-18009.jpg`, `GY-GPS6MV2.jpg`

**推奨**: 区切り文字は1〜2個までとし、過度に複雑な命名は避けます。

#### 4. camelCase の使用

複合語を表現する場合、camelCaseを使用できます。

**例:**
- `breadBoardS.jpg`
- `microbitBreakout.jpg`
- `GroveWaterLevel.jpg`
- `m5QRscanner.jpg`

#### 5. 数字の使用

デバイス型番に数字が含まれる場合は、そのまま使用します。現在、76個のファイルが数字を含んでいます。

**例:**
- `A4988.jpg`
- `BME280.jpg`
- `MPU9250_.jpg`

### 命名例

#### 良い例
- `BME280.jpg` - 型番をそのまま使用
- `HT16K33_7seg.jpg` - 型番とバリエーションをアンダースコアで区切る
- `Grove-Light.jpg` - ブランド名をハイフンで区切る
- `microbit.jpg` - 一般的な名称を小文字で

#### 避けるべき例
- `bme280.jpg` - 型番は大文字で始めるべき
- `HT16K33-7seg.jpg` - 型番のバリエーションにはアンダースコアを使用
- `device_image.jpg` - デバイス名を直接使用すべき

## ファイルサイズ

### 推奨サイズ範囲

- **推奨**: 10KB ～ 100KB
- **最小**: 5KB以上（画質を維持するため）
- **最大**: 300KB以下（リポジトリサイズと読み込み速度のため）

### 現在のサイズ分布

- **小さい（<10KB）**: 21個
- **中（10-100KB）**: 78個（大部分）
- **大きい（>100KB）**: 1個（max30102.webp: 278KB）

### サイズ最適化の推奨事項

1. **画像圧縮**: 適切な圧縮率で画質とファイルサイズのバランスを取る
2. **解像度**: Web表示に適した解像度（通常800x600px程度）を維持
3. **大きなファイル**: 100KBを超える場合は、圧縮や形式変更（WebPなど）を検討

## 画像形式

### 推奨形式

- **`.jpg`**: 主要形式として推奨（現在95個が使用）
  - 写真や複雑な画像に適している
  - 広くサポートされている

### その他の形式

- **`.jpeg`**: `.jpg`と同等だが、統一のため`.jpg`を推奨（現在3個）
- **`.webp`**: より高い圧縮率が必要な場合に使用可能（現在1個）
- **`.png`**: 透明背景が必要な場合のみ使用

### 拡張子の大文字小文字

**重要**: 拡張子は小文字（`.jpg`）で統一してください。

現在、1個のファイル（`AMG8833.JPG`）が大文字拡張子を使用していますが、今後は小文字に統一してください。

### 形式の選択ガイド

| 用途 | 推奨形式 | 理由 |
|------|---------|------|
| 一般的なデバイス写真 | `.jpg` | バランスの良い圧縮率 |
| 高圧縮が必要 | `.webp` | より小さなファイルサイズ |
| 透明背景が必要 | `.png` | 透明度サポート |

## 注意事項

### 既存ファイルの命名不統一について

現在、以下のような命名の不統一が存在します：

1. **拡張子の大文字小文字**: `AMG8833.JPG`（大文字）が1個存在
2. **拡張子の種類**: `.jpg`（95個）、`.jpeg`（3個）、`.webp`（1個）が混在
3. **命名規則**: 大文字始まり（73個）と小文字始まり（27個）が混在

これらは既存ファイルのため、互換性を保つためにそのまま維持されています。

### 今後の追加時の推奨事項

新しい画像ファイルを追加する際は、以下のルールに従ってください：

1. **命名**: デバイス型番または一般的な名称を使用
2. **拡張子**: `.jpg`（小文字）を推奨
3. **ファイルサイズ**: 10KB ～ 100KBを目安に
4. **形式**: 一般的なデバイス写真は`.jpg`を使用
5. **区切り文字**: 必要に応じてアンダースコア（`_`）またはハイフン（`-`）を使用、ただし過度に複雑にしない

### ファイル名の変更について

既存のファイル名を変更する場合は、以下の点に注意してください：

- ドライバーコードやドキュメント内での参照を更新する必要がある
- 変更前に影響範囲を確認すること
